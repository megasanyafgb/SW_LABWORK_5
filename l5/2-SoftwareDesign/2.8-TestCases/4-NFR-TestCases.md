# Тестові сценарії для нефункціональних вимог інтерфейсу користувача

## NFR1.1 - Форма встановлення посади працівника

| NFR ID | TC ID | Опис кроків тестового сценарію | Опис очікуваних результатів |
|--------|-------|--------------------------------|---------------------------|
| NFR1.1 | TC2.1 | **Початкові умови:** відсутні<br/>**Кроки сценарію:** відсутні | Екран повинен відображати форму з наступними елементами:<br/>- Заголовок: "Встановлення посади працівника"<br/>- Поле введення: "Номер працівника"<br/>- Поле введення: "Нова посада"<br/>- Кнопка: "Зберегти"<br/>- Кнопка: "Скасувати"<br/>- Повідомлення про помилку (приховане) |
| NFR1.1 | TC2.2 | **Початкові умови:** успішно завантажена форма (TC2.1 пройдений)<br/>**Кроки сценарію:**<br/>1) Натиснуто поле "Номер працівника"<br/>2) Введено значення "5"<br/>3) Натиснуто поле "Нова посада"<br/>4) Введено значення "Менеджер" | Екран повинен відображати:<br/>- Поле "Номер працівника" містить значення "5"<br/>- Поле "Нова посада" містить значення "Менеджер"<br/>- Кнопка "Зберегти" активна (сіра → яскраво-синя)<br/>- Кнопка "Скасувати" активна |
| NFR1.1 | TC2.3 | **Початкові умови:** успішно заповнена форма (TC2.2 пройдений)<br/>**Кроки сценарю:**<br/>1) Натиснуто кнопку "Зберегти" | Екран повинен відображати:<br/>- Повідомлення про успіх: "Посада успішно встановлена"<br/>- Форма закривається через 2 секунди<br/>- Список працівників оновлюється |
| NFR1.1 | TC2.4 | **Початкові умови:** форма заповнена з невалідним номером працівника (TC2.2 з номером 9999)<br/>**Кроки сценарю:**<br/>1) Натиснуто кнопку "Зберегти" | Екран повинен відображати:<br/>- Повідомлення про помилку: "Номер працівника не знайдений"<br/>- Повідомлення відображається червоним кольором<br/>- Форма залишається відкритою<br/>- Поля можуть бути відредаговані |
| NFR1.1 | TC2.5 | **Початкові умови:** форма заповнена з невалідною посадою (TC2.2 з посадою "менеджер")<br/>**Кроки сценарію:**<br/>1) Натиснуто кнопку "Зберегти" | Екран повинен відображати:<br/>- Повідомлення про помилку: "Посада повинна починатися з великої літери"<br/>- Повідомлення відображається жовтим кольором<br/>- Форма залишається відкритою<br/>- Поле "Нова посада" виділено червоною рамкою |
| NFR1.1 | TC2.6 | **Початкові умови:** форма відкрита<br/>**Кроки сценарію:**<br/>1) Натиснуто кнопку "Скасувати" | Екран повинен:<br/>- Закрити форму<br/>- Повернутися до попередньої сторінки<br/>- Зміни не зберігаються |
